##
## ULTRAFRENCH DICTIONARY
##
## Note: this file uses a custom file format that is MOSTLY LaTeX, but not
## entirely! It is processed by a separate program known as the ULTRAFRENCHER,
## which converts this into both actual LaTeX and JSON+HTML. All macros used in
## this file must also be defined in the ULTRAFRENCHER.
##
## Lines starting with ‘#’ as well as empty lines are ignored. Entries are
## automatically sorted alphabetically; they can be in any order in this file.
##
## By default, each line in this file is an entry in the dictionary.
## To break an entry across multiple lines, indent the continuation lines
## with four spaces. Line-breaking does not affect how the text is displayed.
##
## An entry consists of several fields, separated by a single ‘|’ character. No
## field may contain a ‘|’ character. The fields are as follows:
##
##     1. The word in the UF orthography.
##
##     2. The part of speech and other initial annotations.
##
##     3. The etymology of the word. If this field contains no spaces, it
##        is assumed to be a PF word.
##
##     4. The definition of the word. This may consist of a primary sense, and
##        several additional senses, each of which start with the ‘\\’ macro.
##        Both are optional, but at least one is required.
##
##        A comment and any number of examples, in that order!, may be added to a
##        sense or the primary def using the ‘\comment’ and ‘\ex’ macros. Neither macro
##        takes an argument; they server only as delimiters.
##
##     5. (optional) Grammatical forms of the word.
##
##     6. (optional) The pronunciation of the word in IPA. This should be omitted unless the
##        word is not pronounced as spelt as it will be generated from the spelling otherwise.
##
##        Do NOT wrap the pronunciation in // or []
##
## EXAMPLE:
##
## dír|v. tr.|dire|+\s{acc} To say, tell (+\s{dat} someone) |\s{fut} dírẹ́, \s{subj} díss
## ^^^ ^^^^^^ ^^^^ ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^ ^^^^^^^^^^^^^^^^^^^^^^^^^^^
## 1   2      3    4                                         5
##
## In addition to regular entries, it is possible to create references to other
## entries by adding a line that contains NO ‘|’ characters and is of the form
##
##     entries > reference
##
## where ‘entries’ is a comma-separated list of words for which to create
## references, and ‘reference’ is the word to which the references should
## point.
##
## As in the rest of the grammar, use ‘\s{...}’ for small-caps.
##
## In entries, the following formatting conventions apply.
##
##   - If the etymology is a single PF word (w/o spaces), just put it in
##     the field by itself, and it will be typeset accordingly and in italics.
##
##   - Use ‘\w’ and ‘\w’ only to refer to other UF words within an entry
##
##   - Do not use any markup in the ‘forms’ field (except ‘\s’ for making ‘fut’ etc.
##     italic). FIXME: Do that in the ULTRAFRENCHER instead.
##
## A `$backend` directive (search for occurrences in this file to see how it’s used)
## can be used to restrict a set of entries to the printed or online dictionary.
